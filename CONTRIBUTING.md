# Contributing to Pocket PDF

Thanks for helping improve Pocket PDF.

## Before You Start

- Check existing issues and pull requests first.
- Keep changes focused and easy to review.
- If you are making a larger feature, open an issue before starting implementation.

## Local Setup

1. Install Android Studio and the Android SDK.
2. Use JDK 17.
3. Open the project.
4. Let Gradle sync finish.
5. Build with `./gradlew assembleDebug`.

## Contribution Guidelines

- Use clear commit messages.
- Preserve the app's offline-first behavior.
- Avoid unrelated refactors in feature PRs.
- Test the changed flows before submitting.
- Update documentation when behavior changes.

## Pull Request Checklist

- The app builds successfully.
- User-facing changes were tested.
- README or docs were updated when needed.
- Screenshots or short notes were added for UI changes.

## Questions

If something is unclear, open an issue and describe the problem or idea.
